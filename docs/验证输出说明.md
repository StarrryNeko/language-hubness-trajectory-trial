# 验证输出说明

运行：

```bash
python src/run_pilot.py --config configs/qwen25_1_5b_mvp.json
```

总启动脚本会依次准备数据、提取表示、计算主指标、绘图、运行 `k=5/10/20`
稳健性分析，并在 `outputs/<experiment>/validation/` 中分别保存以下验证结果。
数据准备阶段会先扫描已知异常后缀（当前包括 `.x`）；发现后写入
`dataset_manifest.json` 并在模型提取前停止，避免错误数据继续消耗 GPU。

| 文件 | 验证问题 | 主要判定依据 |
| --- | --- | --- |
| `01_dataset.md/json` | 平行数据是否完整 | 行数、语言均衡、完整语义组、重复键与重复文本 |
| `02_sentence_token_audit.md/json` | 每个 hidden-state 是否能追溯到原句；末 token 是否受标点影响 | 原句、完整 token 序列、末字符、原始末 token、末内容 token、统一 sentinel |
| `02_terminal_character_distribution.csv` | 其他语言是否也以英文句号结尾 | 各语言末字符频数及语言内占比 |
| `02_suspicious_terminal_rows.csv` | 是否存在 `.x` 等异常数据后缀 | 字母/数字结尾与已知可疑后缀的逐句清单 |
| `03_hidden_integrity.md/json` | hidden-state 文件是否正确 | shape、metadata 行数、NaN/Inf 抽查、相邻层重复、截断 |
| `04_semantic_alignment.md/json` | 表示是否包含跨语言句义 | 多次 derangement AlignmentGain CI、Recall@1 CI 与连续层段 |
| `05_english_specificity.md/json` | 英语是否高于所有伪锚点 | 按 semantic ID 聚类 bootstrap 的 `EN − pseudo-anchor` CI |
| `06_english_attraction_hubness.md/json` | 英语邻居吸引和经典 hubness 是否成立 | 邻居语言占比、方向不对称 CI、点级 k-occurrence |
| `07_language_reseparation.md/json` | 末层是否相对中层重分化 | 排除 layer 0 后，预设 late-window − mid-window purity CI |
| `08_representation_robustness.md/json` | 结论是否依赖末 token 或 pooling | 五种表示的结论支持、峰值层及几何有效性 |
| `09_k_robustness.md/json` | 结论是否依赖 k | k=5/10/20 的结论支持与峰值层变化 |
| `validation_summary.md/json` | 是否可以扩大实验 | 汇总状态与放行规则 |

## 为什么不能只看原始 last-token

对 causal decoder 而言，原始最后位置已经看过完整左侧句子，因此适合作为句子表示候选；
但在自然文本中它经常是句末标点。不同语言可能共享英文句号 `.`，也可能使用中文句号 `。`、
阿拉伯问号等不同 token。此时 layer 0 和部分浅层差异可能主要来自标点 embedding、书写系统或
tokenizer，而不是句义。

因此本项目保留原始 `last_token` 作为历史基准，但必须同时比较：

1. `last_content_token`：移除句末标点影响；
2. `shared_sentinel`：所有语言在句尾追加同一个 token，该位置看过完整句子且 token 身份一致；
3. `mean_pool`：保留原始 pooling 对照；
4. `content_mean_pool`：排除特殊 token 和纯标点 token。

只有当核心结论在 `last_token`、`last_content_token` 和 `shared_sentinel` 间方向一致，且没有
低方差/各向异性警告时，才应称为对末 token 选择稳健。

## 查看任意 hidden-state 对应的句子

```bash
python src/inspect_hidden_states.py \
  --config configs/qwen25_1_5b_mvp.json \
  --rows 0,1,2 \
  --layers 0,16,28 \
  --representations last_token,last_content_token,shared_sentinel \
  --show-token-sequence
```

该命令会显示原句、逐 token 内容、三个关键 token 位置，以及所选层向量的 norm、均值、
标准差和前若干维。它不需要重新加载模型。
