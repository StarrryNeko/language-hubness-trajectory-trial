# Sentence-level Pilot 操作与验证清单

## 0. 今日目标

今天需要完成的不是直接证明“模型用英语思考”，而是确认整套 Pilot 是否能够可靠回答以下问题：

1. FLORES 平行数据是否正确、完整、语义对齐；
2. 每层读取到的 hidden state 是否包含可辨别的完整句子语义；
3. cosine similarity 是否计算正常，平行句和非平行句是否能拉开差距；
4. 跨语言靠近是否特别偏向英语，而不是所有语言都相互靠近；
5. 英语是否具有跨语言 kNN Hub 优势和方向不对称；
6. 最后几层是否出现语言重分化；
7. 结果是否依赖 pooling 方法或 k 值；
8. 当前证据允许得出什么结论，哪些结论仍不能得出。

## 7. 每层句子语义是否成立

核心输出：

```text
metrics/semantic_retrieval.csv
metrics/alignment_gain.csv
figures/semantic_retrieval_recall1.png
figures/alignment_gain_by_layer.png
```

### 7.1 跨语言平行句检索

每语言 200 句时，Recall@1 随机水平为：

```text
1 / 200 = 0.005 = 0.5%
```

确认：

- [ ] 至少部分中间层 Recall@1 明显高于 0.5%；
- [ ] Recall@5 高于对应随机水平；
- [ ] 多个语言方向都高于随机水平；
- [ ] 不是只有 EN→ZH 单一方向有效；
- [ ] 置信区间支持检索能力高于随机基线；
- [ ] 能定位语义检索开始明显上升的层；
- [ ] 能定位语义检索峰值层。

解释规则：

- Recall 明显高于随机水平：hidden state 包含可辨别的完整句子语义；
- Recall 接近随机水平：当前表示可能不适合作为句子表示，或该层尚未形成稳定语义；
- 只有 mean pooling 有效：last-token 可能受句末 token 影响过强；
- 只有 last-token 有效：mean pooling 可能混合了过多局部、未看完全文的 token 表示。

### 7.2 AlignmentGain

定义：

```text
AlignmentGain = 平行句 cosine - 打乱语义配对后的 cosine
```

确认：

- [ ] 平行句平均相似度高于 shuffled 相似度；
- [ ] AlignmentGain 在部分层显著大于 0；
- [ ] 95% bootstrap CI 下界高于 0；
- [ ] 多数语言对呈现类似趋势；
- [ ] 峰值层与 Recall@1 较高层段基本吻合。

如果 raw cosine 很高但 AlignmentGain 接近 0，不能解释为语义对齐，更可能是空间整体收缩或各向异性。

## 8. 相似度计算可靠性验证

检查：

```text
metrics/sanity_checks.csv
figures/similarity_sanity_check.png
```

### 8.1 数学 sanity check

- [ ] Self cosine 接近 1；
- [ ] cosine 没有超过 [-1, 1] 合理误差范围；
- [ ] 没有 NaN 或 Inf；
- [ ] `sim(x,y)` 与 `sim(y,x)` 对称；
- [ ] off-diagonal similarity 有非零方差；
- [ ] L2 normalization 正常执行。

### 8.2 三类相似度关系

总体应观察：

```text
Self similarity > Parallel similarity > Shuffled similarity
```

确认：

- [ ] Self similarity 最高；
- [ ] Parallel 与 Shuffled 曲线能够拉开；
- [ ] 差距不是只由某一个语言对贡献；
- [ ] 差距在 bootstrap 下稳定；
- [ ] last-token 与 mean pooling 的方向基本一致。

### 8.3 是否需要进一步调整相似度

出现以下情况时，需要增加 layer-wise mean centering 稳健性版本：

- 所有非对角 cosine 都接近 1；
- Parallel 与 Shuffled 几乎重合；
- off-diagonal standard deviation 极低；
- kNN 结果被少数全局中心向量垄断；
- 不同语言间相似度无法拉开，但检索也接近随机。

建议校正方式：

```text
z_centered = z - 当前层全部样本的均值
z_normalized = L2Normalize(z_centered)
```

注意：

- 原始 cosine 必须保留为主结果；
- centered cosine 只作为稳健性分析；
- 如果校正前后结论相反，应报告结果对几何校正敏感；
- 不应在看到结果后只选择更符合预期的版本。

## 10. 英语 Hubness 与方向不对称

检查：

```text
metrics/neighbor_direction_matrix.csv
metrics/english_directional_asymmetry.csv
figures/english_hub_attraction_by_layer.png
```

四语言实验中，排除 query 自身语言后，英语邻居的均匀基线为：

```text
1 / 3 ≈ 0.333
```

确认：

- [ ] ZH、DE、HA query 的英语近邻比例是否超过 1/3；
- [ ] 95% CI 下界是否超过均匀基线；
- [ ] 英语优势是否出现在连续层段，而不是单个噪声层；
- [ ] `P(EN|X) > P(X|EN)` 是否成立；
- [ ] 英语方向不对称是否对多个非英语语言成立；
- [ ] Hub 峰值层与 Specificity 峰值层是否接近；
- [ ] 正确翻译已经从 Hubness 候选池排除；
- [ ] 各候选语言样本数相同。

解释规则：

- Specificity > 0，但 Hub Attraction 不超过基线：英语方向接近，不支持英语 Hub；
- Hub Attraction 超基线，但方向对称：可能是共享中心或双向对齐；
- Specificity、Hub Attraction和方向不对称共同成立：支持英语偏置中心化；
- 只有单层显著：先按探索性结果处理。

### 11.1 Neighborhood Purity

四语言均衡候选池随机基线约为：

```text
1 / 4 = 0.25
```

寻找：

```text
早期较高 → 中间下降 → 最后几层回升
```

确认：

- [ ] 中间层语言纯度下降；
- [ ] 最后层纯度高于中间最低点；
- [ ] Re-separation Strength 的 95% CI 下界大于 0；
- [ ] 分别检查 EN、ZH、DE、HA；
- [ ] 不是只有语言平均值回升；
- [ ] Centroid Separation 后期也呈上升趋势；
- [ ] 重分化趋势在两种 pooling 下方向一致。

解释规则：

- Purity 和 Centroid Separation 都回升：较强的语言重分化证据；
- 只有英语吸引下降，Purity 不回升：只能说英语偏向减弱；
- 只有 Centroid Separation 回升：证据较弱，可能受全局几何影响；
- 部分语言 CI > 0：报告 partial re-separation；
- 所有语言 CI > 0：可以报告较稳定的语言重分化。

不要仅凭相关曲线使用“自我修正机制”或“correction mechanism”；该说法需要后续干预实验。

- [ ] 大量句子被截断；
- [ ] Recall接近随机水平；
- [ ] Parallel与Shuffled无法分开；
- [ ] 所有cosine接近1且方差极低；
- [ ] 两种pooling给出相反结论；
- [ ] k值改变后结论完全反转；
- [ ] 英语不高于任何伪锚点，却被直接解释为英语中心化。

对应处理：检查数据、表示读取位置、特殊token、空间各向异性和候选池构造，不要直接扩大模型或样本。
